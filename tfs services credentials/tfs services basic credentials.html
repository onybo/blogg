Accessing Team Foundation Services Git from the command line

Since the Visual Studio 2012 Update 2 we have had GIT support right inside the IDE we all know and love. If you aren't familiar with the GIT support in Visual Studio there is a quick introduction on channel 9, and there are several blog posts descibing it. Here's a few: Visual Studio ALM + Team Foundation Server Blog, Jon Galloway and Scot Hanselman.

I have been using the Visual Studio Tools for Git with a GIT repository on Team Foundation Service on a project for customer for some time, and I must say that I have really come to love the GIT source control (have been a TFS user for many years)

Still, Visual Studio Tools for Git is in preview. It is has yet to reach a 1.0 version. There have been some bumps here and there, where we have had the need to go into the command line to fix things. We soon discovered that before we could do so we had to find out how to log into our TFS Git repository, as this was not as simple as you might think. It turned out there is a little tweak needed before you can do that.

Not everyone likes an @ in the username

If you create a new project on Team Foundation Service for the first time, choose GIT sourcecontrol and try to access it from the command line using your Windows Live ID you will probably get the same result as I did:

 

The reason for this is that the git command line tool gets confused when asking for the password for the user: Yourname@somemail.com@yourtfs.visualstudio.com. Notice that there are two @ in the string.

Getting a username without a @

The solution is to get a username without an @. Luckily this is very easy to do. Log into your Team Foundation Service project using your favorite browser. Click your username in the right hand corner as shown below:

 

Click the 'CREDENTIALS' tab in the User Profile dialog that pops up, then click the 'Enable alternate credentials' as shown below:

 

Enter a username without @ and password and hit 'Save changes' and you should be all set to start using command line tools with your new Team Foundation Service project using Git source control
